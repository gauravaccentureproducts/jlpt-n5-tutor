## Native-teacher review notes

### META topics to convert before authoring grammar.json entries

These four entries from JLPT Sensei are conceptual topic-headers, not single-pattern entries. The N4 build should:
- **「他動詞 & 自動詞」** — convert to a reference page or split into individual transitive/intransitive pair entries (e.g., 開ける/開く, 閉める/閉まる).
- **「意向形」** — already covered by `〜よう/〜おう` volitional pattern. Don't double-list.
- **「受身形」** — already covered by `〜られる` passive pattern. Don't double-list.
- **「て / で」** — connector basics already covered at N5; the N4 use is in chains and `てから / てあとで` constructions.

### Patterns whose tier should be `late_n4` or `n3_borderline` (per §A.7 rule)

Borderline / advanced — these should NOT be in `core_n4`:
- `かしら` — feminine register; common in N3 conversation textbooks.
- `さ` (nominalizer suffix) — light N4 / heavy N3.
- `〜させられる` (causative-passive) — Genki II Lesson 23 (last). Use tier=late_n4.
- `〜と言ってもいい` — N3 starting territory.
- `〜させていただけませんか` — formal N3 territory.
- `〜づらい` — compositional with 〜やすい / 〜にくい; N3 in some sources.

### Pattern groups to consolidate

Several patterns are fine-grained variants of a single concept; the build should consider whether each merits its own `n4-NNN` ID or whether they group:

- `みたいだ / みたいな / みたいに` — three forms of one pattern, three IDs OK with shared family
- `ようだ / ように / ような` — same
- `そうだ [1] / そうだ [2] / そうに / そうな` — these are TWO distinct patterns (hearsay vs. appearance) with multiple forms each. CRITICAL to split correctly so questions don't conflate them.
- `〜なければいけない / 〜なければならない` — register variants (casual / formal) of the same obligation pattern.
