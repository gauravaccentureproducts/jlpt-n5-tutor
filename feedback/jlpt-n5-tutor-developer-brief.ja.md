# JLPT N5 学習ツール — 開発指示書（日本語版）

**プロジェクト:** `https://gauravaccentureproducts.github.io/jlpt-n5-tutor/`

**現状:** 静的 SPA（GitHub Pages 配信）、ハッシュベースのルーティング、タブ構成は学習 / テスト / ドリル0 / 復習 / サマリー、ふりがなトグル、端末完結型（バックエンド・分析機能なし）。

**本作業の目的:** 既存の枠組みを、JLPT N5 受験対策として教育的に完結したものへ拡張する。語彙・文法・読解・**聴解**の四領域すべてを実試験レベルでカバーしつつ、静的・プライバシー保護・端末完結という方針は維持する。

**この日本語版について（2026-05-01 追記）:** 本書は元の英語版（`jlpt-n5-tutor-developer-brief.md`）の翻訳である。翻訳の目的は、日本国内のレビュアー（日本語教師、教育研究者、JLPT 関係者）が本プロジェクトの方針・範囲を母語で確認できるようにすることにある。技術実装の詳細は英語版を参照されたい。教育的判断や日本語の妥当性に関わる節は本翻訳でも完訳した。

---

## 0. 譲れない制約条件

以下は変更不可。すべての修正後も成立していなければならない:

1. **完全静的サイト。** GitHub Pages 等のサーバーレス環境で動作し続けること。
2. **データを端末外に出さない。** 分析機能・テレメトリ・第三者トラッキング・通常使用時のリモート API 呼び出しは一切なし。学習進度・SRS 状態・設定はすべて `localStorage`（必要なら `IndexedDB`）に保存する。
3. **ログイン・アカウント不要。** デフォルトで匿名利用可能。
4. **オフライン対応。** 初回読み込み後はオフラインで完全動作。Service Worker と PWA マニフェストを実装する。
5. **クロスブラウザ。** Chrome / Firefox / Safari / Edge の最新二バージョン。モバイル Safari と Chrome Android も一級対応。
6. **バックアップは export/import。** クラウド同期がないため、ユーザーが進度を JSON ファイルに書き出し・再読込できる仕組みを必須とする。

## 1. カリキュラム範囲（収録すべき内容）

### 1.1 文法（目標: N5 全体、約 80〜100 パターン）

学習モジュールには少なくとも以下のパターン群を収録する。順序は以下の通り（『げんき I』と『みんなの日本語 I』の和集合に近い）:

- 名詞文: `Xはです`、`Xじゃありません`、過去 `でした` / `じゃありませんでした`
- 助詞（導入順）: `は`、`も`、`を`、`が`、`に`（存在の場所・時点・方向・間接目的語）、`で`（動作の場所・手段）、`へ`、`と`（共同・列挙）、`から`〜`まで`、`や`（非網羅列挙）、`の`（所有・修飾）、`か`（疑問・選択）
- 指示詞: 完全な「こそあど」表 — `これ／それ／あれ／どれ`、`この／その／あの／どの＋N`、`ここ／そこ／あそこ／どこ`、`こちら／そちら／あちら／どちら`
- 形容詞: い形容詞・な形容詞、肯定／否定／過去／過去否定、名詞修飾の挙動
- 動詞: 三グループ分類（一段／五段／不規則）、辞書形・ます形・ない形・た形・て形、丁寧体・普通体（読みのみで可）
- て形構文: `〜てください`、`〜ています`（進行・状態）、`〜てから`、`〜てもいいです`、`〜てはいけません`
- 助詞構文: `〜があります／います`、`〜が好き／嫌い／じょうず／へた／ほしい／わかる／できる`
- 願望: `〜たいです`、`〜たくないです`、`〜が／をほしいです`
- 移動動詞拡張: `〜に行きます／来ます／帰ります`
- 比較: `AよりBのほうが〜です`、`AとBとどちらが〜ですか`
- 接続詞: `〜から`（理由）、`〜ので`、`〜ましょう／ましょうか`、`〜でしょう／だろう`、`〜けど`、`〜が`（逆接）
- 助数詞: 主要なもの十数種以上
- 疑問詞群: `なに／なん`、`だれ`、`いつ`、`どこ`、`どう`、`なぜ／どうして`、`いくら`、`いくつ`
- 必要表現: `〜なくてはいけない／なくてはならない／なきゃいけない／なくちゃ`
- 推量・比喩: `〜だろう`、`〜らしい`（弱導入）、`〜ようだ`（弱導入）
- 過剰: `〜すぎる`

各パターンは以下を備える:

- パターン記述（日英対訳）
- 例文 4〜6 個（読みがな付き、英訳付き）
- よくある誤りの注釈（少なくとも一つ）
- いつ使うか・いつ使わないかの記述

### 1.2 語彙（約 800 語）

`vocabulary_n5.md` 由来の N5 語彙データを再利用すること。意味カテゴリーごとに分類されているため、その分類を活かしたグルーピングUIを構築する。

### 1.3 漢字（約 100 字）

公式 N5 漢字（106 字 — Pass-13 復元後）のリストを `kanji_n5.md` から取得。各字は以下を表示する: 字形、音読み、訓読み、英語の意味、N5 範囲内の用例（注: 自動生成のふりがな機能は文脈依存読みを誤推定するため Pass-13 で削除済み）。

### 1.4 読解・聴解コーパス

- **読解**: 短文（30〜80 字）、対話、説明文、現実的な構文（メール、お知らせ、簡単な広告）。少なくとも 30 篇。
- **聴解**: JLPT N5 の三形式に倣い構成 — 課題理解、ポイント理解、発話表現。最低 12 件、目標 30 件以上。**重要: 新規収録は必ず日本語母語話者の音声を用いる**（合成 TTS は N5 聴解の音韻・韻律識別力を歪めるため）。

## 2. 構築・改修すべきモジュール

### 2.1 動詞分類モジュール（新規 — 活用導入の前段に置く）

学習者は活用練習の前に、動詞のグループ分類を獲得しておく必要がある。

- 動詞の辞書形を提示し、「一段／五段／不規則」のうちどれかを選ばせる
- 即時フィードバック: 正解／不正解 + 短い理由（語幹規則の説明）
- 進度に応じて動詞プールを拡張
- N5 範囲の典型例外動詞（`帰る／走る／知る／入る／切る／要る` など）を明示する

### 2.2 て形練習モジュール（新規 — 専用ページ）

辞書形を提示 → 学習者が te-form を入力（または複数選択）→ 即時判定。

カバーすべき活用規則:

- う/つ/る → って
- ぶ/ぬ/む → んで
- く → いて、ぐ → いで
- す → して
- 一段動詞 → る を取って て
- 不規則: する → して、来る → きて
- 行く（例外）→ 行って

### 2.3 活用練習場（新規 — て形練習の上位）

ます形・ない形・た形・て形を含む活用練習。学習者が活用形と動詞を選び、答えを入力する。

### 2.4 助数詞モジュール（新規）

主要な助数詞（人・つ・本・枚・冊・回・年・月・日・時・分・歳など）の練習。連濁・連声規則の典型例を可視化する（例: 三本 → さんぼん、六本 → ろっぽん）。

### 2.5 こそあど（指示詞）ページ（新規）

注: ユーザー向け表示は「Demonstratives（this / that / which）」と表記する。「こそあど」は社内・コード内の用語として残し、UI 上には出さない（2026-05-01 利用者方針）。

四系列 × 四種を表形式で提示し、空間関係（話し手 / 聞き手 / 双方から離れた / 疑問）を強調する。

### 2.6 「は」対「が」モジュール（新規または改修）

主題助詞「は」と主格助詞「が」の使い分け。最小対立ペアによる即時練習。

### 2.7 助詞最小対立ドリル

`に` 対 `で` 対 `を` のような典型的な誤用ペアを文脈中で識別させる。

### 2.8 読解モジュール（新規）

短文 → 数問の理解度問題。原文と問題文の間で語彙・文法は N5 範囲を厳守。

### 2.9 聴解モジュール（新規 — 現アプリ最大の欠落）

三形式すべて: 課題理解、ポイント理解、発話表現。音声が必須であり、合成 TTS では JLPT N5 の聴解力を測れないため、**母語話者録音**を用いる。

### 2.10 産出課題（新規）

短い文を学習者に組み立てさせる課題。並べ替え、空欄補充、一語入力。

### 2.11 SRS 復習タブ（既存タブの改修）

**2026-05-01 更新**: SRS スケジューラーを SM-2 から FSRS-4 へ置換した（v1.7.0）。FSRS-4 は Anki ≥23.10 が標準採用するアルゴリズムで、SM-2 より忘却予測の精度が高い。新しいデータ収集は不要で、既存の SM-2 状態は初回 FSRS 評価時に自動的に移行する。四段階評価ボタン（Again / Hard / Good / Easy）の UI は変更なし。

### 2.12 診断サマリータブ（既存タブの改修）

弱点パターンの抽出、復習推奨の生成、学習履歴の可視化。

### 2.13 「Drill 0」改名

意味不明なラベル。「日次ドリル」または「Practice」へ変更（実装済み: 2026-04-30 にナビ簡素化に伴い「Test」のみ残した）。

## 3. 横断的要件

### 3.1 ふりがな表示（`<ruby>` タグ使用）

注: 2026-04-30 に自動ふりがな生成機能は削除された。単一主読み参照テーブルでは文脈依存の読みを正しく選べないため（例: `大学` を `おお` + `がく` と表示してしまう問題）。代替として、N5 漢字は素で表示し、N5 範囲外の漢字は仮名で著者が記述する方針に切り替えた。

### 3.2 言語メタデータ

`<html lang="en">`、日本語スパンには `<span lang="ja">` を付与する。

### 3.3 タイポグラフィ

日本語フォント順は: Noto Sans JP, Hiragino Sans, Yu Gothic, Meiryo, sans-serif。第三者フォント読み込みなし。

### 3.4 アクセシビリティ（WCAG 2.1 AA）

スキップリンク、フォーカス表示、`prefers-reduced-motion`、`prefers-color-scheme`、適切な見出し階層、代替テキスト。

### 3.5 UI 多言語化

英語が一次。`vi` `id` `ne` `zh` の翻訳済み（2026-04-30 OQ-6 決定: アプリの UI と説明は英語のまま、教える内容のみ日本語）。日本語 UI ロケールは作らない。

### 3.6 設定パネル（新規）

音声再生速度、リセット確認、エクスポート／インポート、減モーション設定。

### 3.7 性能

FCP < 1.5s（4G シミュレーション下）、コードスプリットなし（小規模 SPA）、Service Worker による静的キャッシュ。

### 3.8 PWA

マニフェストファイル、アイコン、`apple-touch-icon`、Service Worker による更新トースト。

## 4. データスキーマ（推奨）

### `grammar.json`

各パターンは: `id`、`pattern`、`category`、`meaning_en`、`meaning_ja`、`examples[]`（`ja`、`form`、`translation_en`、`note?`）、`common_mistakes`、`notes`、`requires_patterns?`。

### `vocab.json`

各語は: `id`、`form`、`reading`、`gloss`、`section`、`pos`（2026-05-01 追加: 品詞タグ — noun / verb-1 / verb-2 / verb-3 / i-adj / na-adj / adverb / particle / conjunction / interjection / pronoun / counter / numeral / demonstrative / question-word / expression）。

### `kanji.json`

各字は: `id`、`glyph`、`on[]`、`kun[]`、`meanings[]`、`stroke_order_svg`、`examples[]`（K-1 規則: 各例語は N5 範囲漢字のみ。範囲外漢字を含む語は仮名置換して保存）。

### `srs_state`（`localStorage`）

各パターン ID に対し: `attempts`、`correct`、`incorrect`、`errorRate`、`isWeak`、`isMastered`、`stability`（FSRS）、`difficulty`（FSRS）、`lastReview`、`lapses`、`nextDue`。互換用に SM-2 の `easeFactor`、`interval`、`reps` も保持（実行時には使用しない）。

## 5. 技術推奨（任意）

- バニラ JS（ES6 モジュール）。バンドラー不要。
- CSS のみ。SCSS なし。
- `<dialog>` 要素を使わず、独自モーダル。
- `localStorage` の名前空間プレフィックス: `jlpt-n5-tutor:`。

---

## 6. 受け入れ基準（機能別）

各モジュールに対し、最低限以下を満たす:

- 起動時にコンソールエラーゼロ
- キーボード操作のみで全 UI 到達可能
- スクリーンリーダーで意味のある読み上げ
- ローカルストレージ消去後、初期状態へ復帰
- オフライン状態で機能継続

---

## 7. 推奨実装順序

1. **第一段階**: コーパス整備（`grammar.json`、`vocab.json`、`kanji.json` の充実）
2. **第二段階**: 学習モジュール（文法詳細、語彙閲覧、漢字一覧）
3. **第三段階**: 練習モジュール（ドリル、活用練習、こそあど、は対が、助詞、助数詞）
4. **第四段階**: 産出課題、読解、聴解
5. **第五段階**: SRS 復習、診断サマリー
6. **第六段階**: 設定パネル、PWA、多言語化、A11y 仕上げ

## 10. 開発者へのメモ

- 日本語例文の「自然さ」が最重要。学習者は不自然な日本語を正しく覚えてしまう。母語話者レビューを定期実施。
- 「正しいが不自然」は「間違い」と同等に扱う。
- フォーカスは N5 全範囲の **完璧なカバー**。N4 への前倒しはしない。
- 静的・端末完結という基本方針は譲らない。
